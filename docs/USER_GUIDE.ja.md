## ✨ バージョン 0.33.0 の新機能

- **再起動しても進捗が失われません**：長時間の処理（小説全体の校正や翻訳、大規模なエージェント実行）は、最初からやり直すのではなく**中断した所から再開**します。音声合成もキャッシュされ、1章だけ修正してオーディオブックを再生成しても、変更部分のみが再合成されます。
- **コーダーにセーフティネット**：編集前に自動スナップショットを取得。修正してもテストが失敗し続ける場合、ワンステップで元の状態に巻き戻せます。
- **AI障害の削減**：モデルプロバイダーが飽和（上限到達）した場合、失敗を待たずに呼び出し**前**に健全な代替モデルへ切り替えます — 無料枠がひとつのプールとして機能します。
- **自己学習スキルを監視下に**：新しいスキルはまずテストされ、その後お試し期間（「カナリア」）へ — 実際の使用で複数回成功して初めて正式採用され、失敗すれば除外されます。
- **よりクリーンな記憶**：再確認されない事実は時間とともに薄れ、矛盾が生じた場合（引っ越し、転職など）は最新の事実が優先されます。
- **コックピット → 「🕒 最近の実行」**：過去の実行をクリックするとタイムライン（呼び出したツール、引き継ぎなど）が展開され、**ワンクリックで再実行**できます。
- **セキュリティ強化**：XSS対策を大幅に強化。セッションは使用中に自動延長（作業中のログアウト解消）されつつ永続はせず、SSOトークンはログに残らず、脆弱な依存関係は自動で警告されます。

## 1. 🌟 初回起動とログイン

Athena がお使いのマシンにインストールされ起動すると、グラフィカルインターフェース（UI）はウェブブラウザ経由でのみアクセスできます。

1. ブラウザを開き、次のアドレスにアクセスします：**`http://localhost:8000`**（別のマシンにインストールした場合は、そのサーバーの IP アドレス）。
2. 初回起動の場合、メールアドレスとパスワードでアカウントを作成するよう求められます。
3. ログインすると、**仮想デスクトップ**に到達します。

## 2. 💬 AI と対話する

Athena と対話するには**2 つの方法**があります：ウェブインターフェースとターミナル（CLI）です。

### A. ウェブインターフェース（UI）から
最も視覚的でシンプルな方法です。
- **3D ビュー**：メインページでは、Athena とその「エージェント」（コーダー、リサーチャーなど）の視覚的な表現が見られます。Athena が考えたりタスクを委任したりすると、どのエージェントが作業中かを示す光のアニメーションが表示されます。
- **自動オーケストレーター**：下部のチャットバーに、あなたのリクエストを入力するだけです。ウェブインターフェースでは、**常にオーケストレーターと話している**ことになります。彼はあなたのリクエストを理解し、適切なエージェントに自動的に割り当てられるほど賢いのです（例：スクリプトを求めればコーダーエージェントに引き継ぎます）。
- **チャット内の Artifacts**：AI がプレビュー可能なコード（HTML、**React**、SVG、**Mermaid**、**Markdown**）を生成すると、**「👁️ プレビュー」** ボタンで右側に**ドッキングされたプレビューパネル**が開きます——隔離されたサンドボックスで実行されます。会話の中で生成された**複数のバージョン間を移動**したり、コードを**コピー／ダウンロード**したり、**「🎨 AthenaDesign で開く」** をクリックしてスタジオで続けたりできます。
- **ファイルエクスプローラー（ワークスペース）**：左側にはあなたのファイルを含むパネルがあります。AI が分析できるよう、ドキュメント（PDF、Markdown、ソースコード）をドラッグ＆ドロップできます。
- **内蔵エディター（ミニ IDE）**：ファイルをクリックすると、ブラウザ内で直接**編集**できます——複数のファイルを**タブ**で開け、シンタックスハイライト、オートコンプリート（Ctrl+スペース）、そして **Ctrl+S**（💾）での**保存**が可能です。共有プロジェクトの*閲覧者*は読み取り専用のままです。**エクスプローラーを縮小または折りたたむ**（中央のハンドルまたは「◀ 縮小」ボタン）ことでエディターを広げられ、**エージェントが開いているファイルを変更する**と、あなたのビューは**ライブで更新**されます（他の閲覧者の在席状況とともに）。
- **コーダーコンソール（対話型ターミナル）**：開発のために**コーダー**エージェントと話す、本物のターミナルです。特徴：
  - **独立したターゲットプロジェクト**：セレクターにより、チャット/音声とは**異なる**プロジェクトでコーディングできます（音声コマンドとホームオートメーションはそれぞれのコンテキストで継続します）。
  - 右側の**プロジェクトツリー**（不要なためチャットは非表示）：**自動的に更新**されます——エージェントが作成したファイルが現れるのが見えます。
  - **別ウィンドウの IDE**：**「⧉ IDE」** ボタン（またはツリー内のファイルをクリック）で、エディターが**本物の移動可能なウィンドウ**で開きます（2 つ目の画面に最適）。タブ、ハイライト、オートコンプリート、**Ctrl+S** 付き。*（初回使用時は、サイトのポップアップを許可してください。）*
  - `$` または `!` で始まるコマンドはシェルとして直接実行されます。それ以外の場合、コーダーエージェントがあなたのリクエストを処理し、**ファイルをプロジェクトに書き込みます**（プロジェクトにマウントされた Docker サンドボックス）。

### C. プロジェクトとコラボレーション（共有ワークスペース）
**プロジェクト**は専用の作業フォルダです。プロジェクトバー（エクスプローラー上部）でプロジェクトを選択すると、**AI が行うすべて（読み取り、コード編集、ターミナル、git）はそのフォルダに限定されます**——コードリポジトリやクライアントフォルダを隔離するのに便利です。
- **プロジェクトの作成/切り替え**：`＋ プロジェクト` ボタンを押してからリストで選択。各ユーザーは自分のプロジェクトを持ち、他者には見えません。
- **プロジェクトの共有**（`👥 共有` ボタン）：**所有者**として、他のユーザーを招待し、その役割を選びます：
  - **閲覧者**：プロジェクトを閲覧し AI と会話できますが、**何も変更できません**——エージェントに頼んでも同様です（ロックはサーバー側であらゆる書き込みツールに適用されます：ファイル編集、git、bash、Python）。回避は不可能です。
  - **編集者**：ファイルの変更、コードの実行、コミットができます。
- 共有、役割の変更、プロジェクトの削除ができるのは所有者だけです。

### B. 対話型コンソール（CLI）から
グラフィカルインターフェースよりターミナルがお好みなら、純粋なテキスト対話を開始できます。
Athena のフォルダに移動して次を入力します：
`python3 athena_cli.py`

> [!TIP]
> **特定のエージェントを強制する（コンソールのみ）**：オーケストレーターがすべてを処理するウェブインターフェースとは異なり、コンソールでは指揮者を迂回して専門エージェントと直接話せます。そのためには、次を使います：
> `python3 athena_cli.py --agent Codeur`

## 3. 🛠️ Athena に何ができるのか？（スーパーパワー）

Athena は単なる「ChatGPT」ではありません。それは、あなたのマシンやウェブ上で動作できる数十のツール（「Skills」）を統合した、**自律型人工知能エージェント**のフレームワークです。

### 💻 エージェント型コード（ソフトウェアエンジニアリング）
これがシステムの中核です。Athena はシステム管理者や開発者の代わりを務められます：
- **Python & Bash の実行（サンドボックス）**：AI はコードを書き、安全な Docker サンドボックス内で自律的に実行します。
- **その場での Skill 作成**：独自の機能として、AI は自己改善のために新しい「ツール」をコーディングし、それらを基本ソースコードに恒久的に保存できます！
- **SSH 管理**：AI は SSH 経由で他のリモートサーバーに接続し、メンテナンスを行えます。
- **Computer Use（RPA 2.0）**：AI は本物の隠しウェブブラウザを開き、ボタンをクリックし、フォームに入力し、サイトをスクレイピングできます。
- **Git & コードのナビゲーション**：AI はあなたの Git リポジトリを読み、既存のソースコードを理解し、ライブで編集できます（ファイル `glob`／内容検索、ファイルのアウトライン、参照）。
- **編集ごとの診断（フィードバックループ）**：ファイルを変更するたびに、Athena は導入された**エラー／警告**を読み直し（Python は **basedpyright** LSP サーバー、それ以外は内蔵のフォールバック）、**すぐに修正**します。これらの診断は**コード**タブ（「🔍 分析」ボタン）にも表示されます。
- **セッションのタスクリスト**：複数ステップの作業では、エージェントが**チェックリスト**（📋 タスク）を保持し、`athena_cli` とコードタブに表示され、リアルタイムで更新されます。
- **プランモード（読み取り専用）**：**「🧭 プランモード」** ボタン（または CLI の `/plan` / `/build`）——エージェントは**何も変更せずにプランを提案**します。実行するには通常モードに戻します。
- **プロジェクト指示**：プロジェクトのルートに `CLAUDE.md`、`ATHENA.md`、`AGENTS.md` を置くと（規約、コマンド）、Athena が自動的に読み込み、git ルートまでカスケードします。
- **自律メンテナンス**：夜間エージェントがソースコードを自動的にチェックし修復できます。

### 🎨 AthenaDesign Studio（AI デザイン）
内蔵のデザインスタジオ（**🎨 デザイン** タブ）。作りたいものを説明すれば、Athena がそれを生成し、**ライブで表示**します：
- **タイプ**：ウェブページ（HTML/CSS/JS）、インタラクティブな **React アプリ**、**Mermaid 図**、そして **Python** スクリプト（**PowerPoint** プレゼン、グラフ）。**スターターテンプレート**（Landing、Pitch deck、Dashboard…）がプロンプトを事前入力します。
- **あなたのブランド（Design System）**：「Design System」パネルで色／フォントを指定します——手入力、CSS の貼り付け、**「🌐 URL から」**、または**自動生成**：**「🧩 コードから」**（プロジェクトから推定：Tailwind/CSS）、**「🖼️ 画像から」**（スクリーンショットからパレット／タイポグラフィ）、**「✨ 説明から」**（空プロジェクト用の初期ブランド）。
- **参照**：画像／ドキュメント（📎）やウェブページ（🔗）をインスピレーションとして添付します。
- **洗練**：プレビューに注釈を付け、ライブで調整し（色／角丸／フォントのスライダー）、各バージョンを閲覧します。Python スクリプトが失敗すると、Athena は**自分で修正します**。
- **共有／エクスポート**：**共有** ボタン（読み取り専用リンク）、**PDF エクスポート**、`.pptx` のダウンロード。
- *ヒント*：1 つの Athena プロジェクトが**コードとデザイン**をまとめます——両方を同じ場所で管理できます。

### 🔌 プラグイン（Claude Code を含む）
**設定 > 🔌 プラグイン** で拡張機能を有効にします。**Claude Code プラグイン**はコーディングエージェント **Claude Code** を呼び出します（`claude` CLI のインストールとログインが必要）：有効にすると、あなたの**コーダー**は複雑なコーディングタスクを、アクティブなプロジェクト内で直接それに委任できます。*（あなたの Claude サブスクリプション／キーを消費します。）*

### 🏠 ホームオートメーション、コンテキスト、日常
- **ネイティブホームオートメーション（Home Assistant）**：AI はあなたのホームオートメーションに直接接続します。*「リビングの電気を消して」* や *「ブラインドを閉めて」* と頼めば、即座に実行します。
- **MCP 拡張（上級）**：Athena は MCP プロトコルをサポートします。これにより、複雑なプラグイン（オートメーション作成のための Home Assistant データベースへの深いアクセス、またはその他の既存の MCP サーバーなど）を接続できます。
- **空間認識**：AI は（センサーがあれば）あなたがどの部屋にいるかを知り、行動を適応させられます（例：*「電気をつけて」* はあなたがいる部屋の電気をつけます）。
- **天気と時間**：複数日の天気予報と時刻同期。
- **リストと買い物**：買い物リストに牛乳を追加させたり、ToDo リストを作成させたりできます。

### 📅 生産性とコミュニケーション
- **カレンダーと予定管理**：あなたのカレンダー（iCal、CalDAV）と同期して、イベントの読み取りと作成を行います。
- **会議の要約**：会議や音声ファイルを書き起こして要約する能力。
- **通知**：Athena は自らの判断で Telegram、Discord、Slack 経由でメッセージを送れます。
- **メディア生成**：画像の作成（Fal/Replicate API 経由）とファイル操作（PDF、ドキュメント）。
- **ワークフロー（n8n）**：n8n の webhook 経由で複雑なシナリオを起動。

### ⏰ 能動的ルーティン
Athena はあなたが話しかけるのを待ちません。*「毎朝 7:30 にその日の要約を作って」* と頼んでください。自分で目覚め、あなたのカレンダー、天気、家の状態を分析し、コーヒーメーカーを起動することさえできます！

## 4. ⚙️ インターフェースの設定を理解する

サイドバーの歯車アイコン（⚙️）をクリックすると、あなたのプロフィール設定にアクセスできます。**各設定はあなたのユーザーに厳密に隔離されています。**

### 「🔑 API キー」タブ（マイモデルと LLM キー）
- **AI プロバイダー（例：OpenAI、Anthropic、Ollama）** と **モデル名**：使用したい AI のバージョンを選びます。
- **個人 API キー**：このフィールドに入力すると、Athena はあなたのキーで動作し、あなた自身の開発者アカウントに課金されます。これにより、サーバーのデフォルトモデルを上書きできます。
- **📊 マイ使用状況**：キーのすぐ下に、あなた個人の消費（リクエスト、トークン、コスト €）の本日・過去 30 日・累計のまとめがあり——支出を一目で追えます。（管理者は一方で、全アカウントの消費を見られます。）

### 🔐 アカウントを保護する（「ユーザー」タブ）
- **マイパスワード**：いつでも変更できます（最低 8 文字）。セキュリティのため、パスワードを変更すると**他のセッションがログアウト**されます。
- **二要素認証（2FA）**：**2FA を有効化** をクリックし、表示されたシークレットをスキャン／入力して認証アプリ（Google Authenticator、Authy、FreeOTP…）にアカウントを追加し、コードを入力して確認します。以後、ログインのたびにパスワードに加えて**一時コード**が求められます。いつでも無効化できます（確認にコードが必要）。
  - *デバイスを紛失？* 管理者があなたの 2FA をリセットしてアクセスを回復できます。

### 「カレンダーと ToDo」タブ
- **メインカレンダー（URL）**：iCal フィード（Google Calendar）のアドレスを貼り付けます。すると AI はあなたの予定を読めるようになります。
- **CalDAV サーバー（URL、ユーザー、パスワード）**：高度なカレンダー（Nextcloud、Synology）を使用している場合、AI はイベントを直接*作成*・*変更*できます。

### 「動作とセキュリティ」タブ（Athena の頭脳）
これは、マシンの全体的な動作とセーフガードを調整する最も重要なセクションです。いくつかのサブセクションに分かれています：

#### 1. 実行とセーフガード
- `コード/コマンド実行サンドボックス`：AI がスクリプトを安全なサンドボックスで実行するように **Docker**（推奨）を選ぶか、OS 上で直接動作させたい場合は **ローカル** を選びます。
- `自己改善`：AI が失敗から教訓を得て、将来の動作ルールを作成することを許可します。
- `予算（時間とトークン）`：金銭的セーフガード。AI がタスクごとに消費できる最大秒数（0 = 無限）または最大トークン数を制限できます。
- `当日コストアラート`：1 日の支出がこの閾値（ユーロ）を超えると、通知を受け取ります。

#### 2. セキュリティ
- `機密ツールを自動承認`：デフォルト（チェックなし）では、AI は「機密」とマークされたツール（例：システムファイルへの書き込み）を使う前に必ず確認を求めます。チェックすると、AI は完全に自律的になります（自己責任で）。
- `管理者パスワード / CORS オリジン`：望まない外部接続を防ぐためのウェブサーバーの保護。
- `セッションの有効期間`：インターフェースからログアウトされるまでの時間（時間単位）（デフォルト：168 時間、つまり 1 週間）。
- `クォータと制限`：システムはあなたの財務を保護します。管理者はユーザーデータベースで 1 日あたりのトークン消費上限を設定できます。
- `保存時の暗号化`：会話と実行トレースは、Fernet（AES-128-CBC + HMAC）経由でデータベース（SQLite）内で暗号化されます。キーはインストールの `.env` に保存されます——**紛失しないでください**（さもないと暗号化された履歴が読めなくなります）。ディスク盗難への真の保護のためには、フォルダの外に保管してください（注入された環境変数 / シークレットマネージャー）。
- `内蔵保護（不可視）`：Athena はログ内であなたの API キーとシークレットを自動的にマスクし（Redaction）、内部ネットワークやクラウドメタデータへのウェブリクエストをブロックする anti-SSRF 保護を組み込んでいます。

#### 3. オーケストレーションとエージェント（上級）
- `LLM ルーティング（Delegation Router）`：オーケストレーターがあなたのメッセージを読み、適切なエージェントを選びます。
- `高速モデル`：ルーティングの判断だけのために非常に高速なモデル（例：`gpt-4o-mini` や `haiku`）を強制でき、AI の反応が機敏になります。
- `フォールバックモデル`：メイン AI の API がクラッシュした場合、Athena はこれらの予備モデルを使おうとします。
- `プロンプトキャッシュ`：長い会話でお金と時間を節約する技術。
- `自己批評`：有効にすると、AI は回答をあなたに送る前に、自分の回答を読み返してチェックします。

#### 4. メモリ
- `事実ベース（Core Memory）`：Athena があなたについて恒久的に学んだすべて（あなたの好み、職業）を一覧表示します。そこから項目を削除できます。
- `ナレッジグラフ`：単純な事実に加え、AI はコンテキストをよりよく理解するために、エンティティ間の関係のネットワーク（「グラフ」）を構築します。
- `N メッセージを超えたら圧縮`：請求額の爆発を避けるため、Athena は N メッセージ後に会話の古い部分を自動的に要約します（デフォルト 40）。
- `直近メッセージを一字一句保持`：Athena は常に直近 N 回の厳密なやり取りを短期記憶に保持します（デフォルト 12）。

#### 5. 表現力豊かな音声
- `音声の感情`：LLM がテキストに `[laugh]`、`[sad]` タグを挿入し、音声エンジンがトーンを適応させます！
- `表現力 TTS サーバーと音声`：サードパーティの音声エンジン（XTTS など）を使用する場合は、ここにその IP アドレスを入力します。

#### 6. 空間認識（在席 / follow-me）
- `現在の部屋の HA エンティティ`：Home Assistant に在席センサーがある場合は、ここでエンティティを指定します（例：`sensor.current_room`）。すると AI はあなたがどの部屋にいるかを知り、そこで適切な電気をつけたり動作を適応させたりします。

#### 7. 自動化（n8n）
- `許可されたワークフロー`：ウェブアドレス（Webhook）へのアクセスを与えることで、Athena を複雑な n8n 自動化に接続できます。

### 設定パネルのその他のタブ
「動作」に加えて、設定サイドバーは他の専門メニューへのアクセスを提供します：

* **「ナレッジ（RAG）」タブ**：ここで、ファイルエクスプローラーに置いたドキュメントを AI に分析（またはパージ）させることができます。
* **「ルーティン」タブ**：自動タスクをスケジュールできます（例：「毎日 7:00 に家の要約を作る」）。これらのルーティンの「Webhook」アドレスを取得したり、単純なタスクの代わりにルーティンが**決定論的ワークフローを起動**する（フォームの「Workflow」フィールド）ようにしたりもできます。
* **「音声サテライト」タブ**：Athena に接続された ESP32 スピーカーを設定できます。
* **「MCP 拡張」タブ**：標準的な外部プラグイン（GitHub コネクタ、Home Assistant コネクタなど）を AI に接続できます。
* **「診断とシステム」タブ**：インストールの健全性（データベース、STT、TTS）をチェックします。音声の不具合時に**音声エンジン（Kokoro）を再起動**する緊急ボタンや、環境全体の**バックアップと復元**オプションがここにあります。
* **「ワークフロー」タブ**：自律モードの代替として**決定論的パイプライン**（エージェントの連鎖、「組み立てライン」型）を作成します——再現可能で監査可能な流れが欲しいときに便利です。下の専用セクションを参照してください。
* **「ユーザー」タブ（管理者）**：管理者であれば、ここで新しい人を招待し、その権限とトークンクォータを管理できます。また、「ユーザー」アカウントが作成した**自動化**（ワークフロー/ルーティン）が実行できるようになる前にそれらを**検証**し、アカウントの **2FA をリセット**（デバイス紛失）し、`GET /api/audit` 経由で**監査ログ**（ログイン、パスワード変更、検証…）を参照できます。

## 5. 💻 サーバーの管理（管理コマンド）

Athena をホストするマシンの管理者であれば、サーバーのライフサイクルを管理する強力なシステムコマンドが利用できます。

### 🍎 Linux & macOS
ターミナルを開きます。主要なコマンドは `athena` です。
- `athena start`：AI をバックグラウンドで起動します（SystemD / LaunchAgent プロセス）。
- `athena stop`：サーバーをクリーンに停止します。
- `athena restart`：アプリケーションを完全に再起動します。
- `athena status`：サーバーがオンラインかどうかを確認します。
- `athena logs`：サーバーの技術ログをリアルタイムで表示します。（終了するには `Ctrl+C`。）

**ソフトウェアの更新：**
ソースコードのフォルダに移動して実行します：`./update.sh`

### 🪟 Windows（PowerShell）
PowerShell を開きます。管理コマンドは `.ps1` で終わります。
- `athena.ps1 start`：サーバーをバックグラウンドタスクで起動します。
- `athena.ps1 stop`：サーバーを停止します。
- `athena.ps1 restart`：プロセスを再起動します。
- `athena.ps1 status`：状態を表示します。
- `athena.ps1 logs`：オーケストレーターの技術コンソールを表示します。

**ソフトウェアの更新：**
ソースコードのフォルダに移動して実行します：`.\update.ps1`
