## 1. 🌟 首次启动与登录

一旦 Athena 在您的机器上安装并启动，图形界面（UI）只能通过您的网页浏览器访问。

1. 打开浏览器并访问地址：**`http://localhost:8000`**（如果您将其安装在另一台机器上，则使用您服务器的 IP 地址）。
2. 如果是首次启动，系统会要求您用一个电子邮件地址和一个密码创建账户。
3. 登录后，您将进入**虚拟桌面**。

## 2. 💬 与 AI 对话

与 Athena 交互有**两种方式**：网页界面和终端（CLI）。

### A. 通过网页界面（UI）
这是最直观、最简单的方式。
- **3D 视图**：在主页面上，您会看到 Athena 及其"智能体"（程序员、研究员等）的可视化呈现。当 Athena 思考或委派任务时，您会看到发光的动画，提示哪个智能体正在工作。
- **自动编排器**：在底部的对话栏中，直接输入您的请求即可。在网页界面中，**您始终是在与编排器对话**。它足够聪明，能理解您的请求并自动分配给合适的智能体（例如，如果您请求一个脚本，它会移交给程序员智能体）。
- **聊天中的 Artifacts**：当 AI 生成可预览的代码（HTML、**React**、SVG、**Mermaid**、**Markdown**）时，**"👁️ 预览"** 按钮会在右侧打开一个**停靠的预览面板**——在隔离沙盒中运行。您可以在对话过程中生成的**多个版本之间导航**、**复制/下载**代码，或点击 **"🎨 在 AthenaDesign 中打开"** 以在工作室中继续。
- **文件资源管理器（工作区）**：左侧有一个包含您文件的面板。您可以将文档（PDF、Markdown、源代码）拖放进去，让 AI 进行分析。
- **内置编辑器（迷你 IDE）**：点击某个文件可直接在浏览器中**编辑**——多个文件以**标签页**形式打开、语法高亮、自动补全（Ctrl+空格），以及用 **Ctrl+S**（💾）**保存**。共享项目的*读者*保持只读。您可以**收窄或折叠资源管理器**（中央手柄或"◀ 缩小"按钮）以放大编辑器，当**智能体修改已打开的文件**时，您的视图会**实时刷新**（并显示其他读者的在线状态）。
- **程序员控制台（交互式终端）**：一个真正的终端，您在此与**程序员**智能体对话进行开发。特点：
  - **独立的目标项目**：通过选择器可以在与聊天/语音**不同**的项目上编程（您的语音命令和家居自动化继续在各自的上下文中运行）。
  - **项目树**位于右侧（聊天在此被隐藏，因为无用）：它会**自动刷新**——您能看到智能体创建的文件出现。
  - **IDE 独立窗口**：**"⧉ IDE"** 按钮（或点击树中的文件）会在一个**真正可移动的窗口**中打开编辑器（在第 2 块屏幕上尤为理想），带标签页、高亮、自动补全和 **Ctrl+S**。*（首次使用时，请允许该站点的弹出窗口。）*
  - 以 `$` 或 `!` 为前缀的命令直接作为 shell 执行；否则程序员智能体会处理您的请求并**将文件写入项目**（挂载在项目上的 Docker 沙盒）。

### C. 项目与协作（共享工作区）
**项目**是一个专用的工作文件夹。当您在项目栏（资源管理器顶部）选择一个项目时，**AI 所做的一切（读取、代码编辑、终端、git）都被限制在该文件夹内**——便于隔离一个代码仓库或一个客户文件夹。
- **创建/切换项目**：`＋ 项目` 按钮，然后从列表中选择。每位用户都有自己的项目，对他人不可见。
- **共享项目**（`👥 共享` 按钮）：作为**所有者**，您邀请其他用户并选择他们的角色：
  - **读者**：可以查看项目并就项目与 AI 对话，但**不能修改任何内容**——即使请求智能体也不行（锁定在服务器端对每个写入工具强制执行：文件编辑、git、bash、Python）。无法绕过。
  - **编辑者**：可以修改文件、运行代码、提交。
- 只有所有者可以共享、更改角色或删除项目。

### B. 通过交互式控制台（CLI）
如果您更喜欢终端而非图形界面，可以启动一个纯文本对话。
进入 Athena 文件夹并输入：
`python3 athena_cli.py`

> [!TIP]
> **强制指定某个智能体（仅限控制台）**：与网页界面由编排器处理一切不同，控制台允许您绕过指挥家，直接与某个专家智能体对话。为此，请使用：
> `python3 athena_cli.py --agent Codeur`

## 3. 🛠️ Athena 能做什么？（超能力）

Athena 不仅仅是一个"ChatGPT"。它是一个**自主人工智能智能体**框架，集成了数十种能在您的机器和网络上行动的工具（"Skills"）。

### 💻 智能体式代码（软件工程）
这是系统的核心。Athena 可以替代一名系统管理员或开发者：
- **Python 与 Bash 执行（沙盒）**：AI 编写代码并在安全的 Docker 沙盒中自主执行。
- **即时创建 Skills**：独有功能，AI 可以编写新"工具"来自我改进，并将它们永久保存到其基础源代码中！
- **SSH 管理**：AI 可以通过 SSH 连接到您的其他远程服务器进行维护。
- **Computer Use（RPA 2.0）**：AI 可以打开一个真正的隐藏网页浏览器，点击按钮、填写表单并抓取网站。
- **Git 与代码导航**：AI 可以读取您的 Git 仓库、理解您现有的源代码并实时编辑（文件 `glob`/内容搜索、文件大纲、引用）。
- **每次编辑后的诊断（反馈循环）**：每次文件更改时，Athena 都会重新读取引入的**错误/警告**（Python 使用 **basedpyright** LSP 服务器，否则使用内置回退）并**立即修复**。这些诊断也会显示在**代码**标签页（"🔍 分析"按钮）。
- **会话任务清单**：对于多步骤工作，智能体会维护一个**清单**（📋 任务），在 `athena_cli` 和代码标签页中可见，并实时更新。
- **计划模式（只读）**：**"🧭 计划模式"** 按钮（或 CLI 中的 `/plan` / `/build`）——智能体**提出计划而不修改任何内容**；切回普通模式即可执行。
- **项目说明**：在项目根目录放置 `CLAUDE.md`、`ATHENA.md` 或 `AGENTS.md`（约定、命令）——Athena 会自动加载，并向上级联至 git 根目录。
- **自主维护**：一个夜间智能体可以自动检查和修复源代码。

### 🎨 AthenaDesign Studio（AI 设计）
一个内置的设计工作室（**🎨 设计** 标签页）。描述您想创建的内容，Athena 会生成并**实时显示**：
- **类型**：网页（HTML/CSS/JS）、交互式 **React 应用**、**Mermaid 图表**，以及 **Python** 脚本（**PowerPoint** 演示文稿、图表）。**起始模板**（Landing、Pitch deck、Dashboard…）会预填一个提示词。
- **您的品牌规范（Design System）**："Design System" 面板用于提供您的颜色/字体——手动输入、粘贴 CSS、通过 **"🌐 从 URL"**，或**自动生成**：**"🧩 从代码"**（从项目推断：Tailwind/CSS）、**"🖼️ 从图像"**（从截图提取调色板/字体）、**"✨ 从描述"**（为空项目生成起始品牌规范）。
- **参考**：附上一张图片/一个文档（📎）或一个网页（🔗）作为灵感。
- **优化**：在预览上标注、实时调整（颜色/圆角/字体滑块）、浏览各个版本。如果某个 Python 脚本失败，Athena 会**自我纠正**。
- **共享/导出**：**共享** 按钮（只读链接）、**导出 PDF**，以及下载 `.pptx`。
- *提示*：一个 Athena 项目同时汇集**代码与设计**——您在同一个地方管理两者。

### 🔌 插件（包括 Claude Code）
在 **设置 > 🔌 插件** 中启用扩展。**Claude Code 插件**调用 **Claude Code** 编码智能体（需要安装并登录 `claude` CLI）：一旦启用，您的**程序员**可以直接在活动项目中将复杂的编码任务委派给它。*（会消耗您的 Claude 订阅/密钥。）*

### 🏠 家居自动化、上下文与日常
- **原生家居自动化（Home Assistant）**：AI 直接连接到您的家居自动化。让它*"关掉客厅的灯"*或*"关上百叶窗"*，它会立即执行。
- **MCP 扩展（高级）**：Athena 支持 MCP 协议。这使它能够接入复杂的插件（例如深度访问 Home Assistant 数据库以创建自动化，或任何其他现有的 MCP 服务器）。
- **空间感知**：AI 可以知道您身处哪个房间（如果您有传感器），以调整其行动（例如，*"开灯"*会打开您所在房间的灯）。
- **天气与时间**：多日天气预报和时间同步。
- **清单与购物**：让它在您的购物清单上加上牛奶，或创建一个待办清单。

### 📅 生产力与沟通
- **日历与计划**：与您的日历（iCal、CalDAV）同步以读取和创建事件。
- **会议摘要**：能够转录并总结会议或音频文件。
- **通知**：Athena 可以主动通过 Telegram、Discord 或 Slack 向您发送消息。
- **媒体生成**：创建图像（通过 Fal/Replicate API）和文件处理（PDF、文档）。
- **工作流（n8n）**：通过 n8n webhook 触发复杂场景。

### ⏰ 主动例程
Athena 不会等您来跟它说话。让它：*"每天早上 7:30 给我做一份当天的摘要"*。它会自己醒来，分析您的日程、天气、家中状态，甚至可以启动咖啡机！

## 4. ⚙️ 理解界面参数

点击侧边栏中的齿轮图标（⚙️），即可访问您个人资料的设置。**每项参数都严格隔离到您的用户。**

### "🔑 API 密钥"标签页（我的模型与 LLM 密钥）
- **AI 提供商（例如 OpenAI、Anthropic、Ollama）** 和 **模型名称**：选择您想使用的 AI 版本。
- **个人 API 密钥**：如果填写了此字段，Athena 将使用您的密钥运行，并在您自己的开发者账户上计费。这可以覆盖服务器的默认模型。
- **📊 我的用量**：就在您的密钥下方，是您个人消耗的汇总（请求数、token、费用 €），涵盖今天、过去 30 天和总计——一目了然地跟踪您的支出。（管理员则能看到所有账户的消耗。）

### 🔐 保护我的账户（"用户"标签页）
- **我的密码**：随时可以更改（至少 8 个字符）。出于安全考虑，更改密码会**注销您的其他会话**。
- **双因素认证（2FA）**：点击 **启用 2FA**，通过扫描/输入显示的密钥将账户添加到您的认证应用（Google Authenticator、Authy、FreeOTP…）中，然后输入一个验证码以确认。此后每次登录时，除密码外还会要求一个**临时验证码**。您可以随时禁用它（需要验证码确认）。
  - *设备丢失？* 管理员可以重置您的 2FA 以恢复您的访问权限。

### "日历与待办"标签页
- **主日历（URL）**：粘贴一个 iCal 订阅源的地址（Google Calendar）。AI 便能读取您的日程。
- **CalDAV 服务器（URL、用户、密码）**：如果您使用高级日历（Nextcloud、Synology），AI 可以直接*创建*和*修改*事件。

### "行为与安全"标签页（Athena 的大脑）
这是调整机器整体行为与安全防护的最重要部分。它分为多个子部分：

#### 1. 执行与防护
- `代码/命令执行沙盒`：选择 **Docker**（推荐），让 AI 在安全沙盒中执行其脚本，或选择 **本地**，如果您希望它直接作用于您的操作系统。
- `自我改进`：允许 AI 从失败中汲取教训，以制定未来的行为规则。
- `预算（时间与 Token）`：财务防护。可以限制 AI 每个任务允许消耗的最大秒数（0 = 无限）或最大 token 数。
- `当日费用警报`：如果当日支出超过此欧元阈值，您将收到通知。

#### 2. 安全
- `自动批准敏感工具`：默认（未勾选）情况下，AI 在使用标记为"敏感"的工具（例如写入系统文件）前总会请求您确认。如果勾选，AI 将变得完全自主（风险自负）。
- `管理员密码 / CORS 来源`：保护网页服务器，防止不需要的外部连接。
- `会话有效时长`：被注销出界面前的时间（小时）（默认：168 小时，即一周）。
- `配额与限制`：系统保护您的财务。管理员可以在用户数据库中设定每日 token 消耗上限。
- `静态加密`：对话和执行轨迹在数据库（SQLite）中通过 Fernet（AES-128-CBC + HMAC）加密。密钥存储在您安装的 `.env` 中——**不要丢失它**（否则加密的历史记录将无法读取），为真正防范磁盘失窃，请将其保存在文件夹之外（注入的环境变量 / 密钥管理器）。
- `内置保护（不可见）`：Athena 会自动在日志中屏蔽您的 API 密钥和机密（脱敏），并集成了反 SSRF 保护，阻止指向您内部网络或云元数据的网络请求。

#### 3. 编排与智能体（高级）
- `LLM 路由（Delegation Router）`：编排器读取您的消息并选择合适的智能体。
- `快速模型`：您可以仅为路由决策强制使用一个非常快的模型（例如 `gpt-4o-mini` 或 `haiku`），使 AI 反应更敏捷。
- `回退模型（Fallback）`：如果您主 AI 的 API 崩溃，Athena 会尝试使用这些备用模型。
- `提示词缓存`：在长对话中节省金钱和时间的技术。
- `自我批评`：如果启用，AI 会在将答案发送给您之前重读并检查自己的回答。

#### 4. 记忆
- `事实库（Core Memory）`：列出 Athena 永久了解到的关于您的一切（您的喜好、您的职业）。您可以在此删除条目。
- `知识图谱`：除了简单事实外，AI 还在实体之间构建关系网络（"图谱"），以更好地理解您的上下文。
- `超过 N 条消息后压缩`：为避免账单失控，Athena 会在 N 条消息后自动总结对话的旧部分（默认 40）。
- `逐字保留的近期消息`：Athena 始终在短期记忆中逐字保留最近 N 轮严格的交流（默认 12）。

#### 5. 富有表现力的语音
- `语音情感`：LLM 在其文本中插入 `[laugh]`、`[sad]` 标记，语音引擎会相应调整语气！
- `富表现力 TTS 服务器与语音`：如果您使用第三方语音引擎（如 XTTS），请在此填写其 IP 地址。

#### 6. 空间感知（在场/follow-me）
- `当前房间的 HA 实体`：如果您在 Home Assistant 上有在场探测器，请在此指明实体（例如 `sensor.current_room`）。AI 便能知道您身处哪个房间，从而打开正确的灯或调整其行为。

#### 7. 自动化（n8n）
- `已授权的工作流`：您可以通过授予 Athena 访问网络地址（Webhooks）的权限，将其连接到复杂的 n8n 自动化。

### 设置面板的其他标签页
除了"行为"之外，设置侧边栏还为您提供其他专门菜单的访问：

* **"知识（RAG）"标签页**：在这里，您可以让 AI 分析（或清除）您放入文件资源管理器中的文档。
* **"例程"标签页**：可用于安排自动任务（例如"每天 7:00 做一份家中摘要"）。您也可以在此获取这些例程的"Webhook"地址，或让某个例程**触发一个确定性工作流**（表单的"Workflow"字段）而非简单任务。
* **"语音卫星"标签页**：可用于配置连接到 Athena 的 ESP32 音箱。
* **"MCP 扩展"标签页**：可将标准的外部插件（例如 GitHub 连接器、Home Assistant 连接器）接入 AI。
* **"诊断与系统"标签页**：检查安装的健康状况（数据库、STT、TTS）。在音频故障时，**重启语音引擎（Kokoro）**的紧急按钮就在这里，以及您整个环境的**备份与恢复**选项。
* **"工作流"标签页**：创建**确定性流水线**（智能体链，"流水线"式）作为自主模式的替代方案——当您想要可复现且可审计的流程时很有用。请参见下文的专门章节。
* **"用户"标签页（管理员）**：如果您是管理员，可以在此邀请新人、管理他们的权限和 token 配额。您还要在"用户"账户创建的**自动化**（工作流/例程）可以运行之前对其进行**验证**，可以**重置某账户的 2FA**（设备丢失），并通过 `GET /api/audit` 查看**审计日志**（登录、密码更改、验证…）。

## 5. 💻 管理服务器（管理命令）

如果您是托管 Athena 的机器的管理员，您拥有强大的系统命令来管理服务器的生命周期。

### 🍎 Linux 与 macOS
打开您的终端。主命令名为 `athena`。
- `athena start`：在后台启动 AI（SystemD / LaunchAgent 进程）。
- `athena stop`：干净地关闭服务器。
- `athena restart`：完全重启应用程序。
- `athena status`：检查服务器是否在线。
- `athena logs`：实时显示服务器的技术日志。（按 `Ctrl+C` 退出。）

**更新软件：**
进入源代码文件夹并运行：`./update.sh`

### 🪟 Windows（PowerShell）
打开 PowerShell。管理命令以 `.ps1` 结尾。
- `athena.ps1 start`：在后台任务中启动服务器。
- `athena.ps1 stop`：停止服务器。
- `athena.ps1 restart`：重启进程。
- `athena.ps1 status`：显示状态。
- `athena.ps1 logs`：显示编排器的技术控制台。

**更新软件：**
进入源代码文件夹并运行：`.\update.ps1`
